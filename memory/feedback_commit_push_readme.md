Na elke succesvolle wijziging aan de app: commit + push + README bijwerken. Dit niet apart hoeven vragen.

**Why:** De gebruiker wil de repo altijd up-to-date houden en de README synchroon met de huidige staat van de app.

**How to apply:** Zodra een feature of fix werkt, direct git add / commit / push uitvoeren én de README aanpassen zonder dat de gebruiker hierom vraagt.
